# Instagram RSS → Telegram репостер

Бот раз в день забирает RSS-фид (сделанный через rss.app) и репостит новые
посты (текст без хештегов + картинка) в Telegram-канал. Работает целиком на
GitHub Actions, без сервера.

## Как это работает

1. `bot.py` скачивает RSS-фид.
2. Сравнивает посты с уже отправленными (хранятся в `posted.json` по `guid`).
3. Новые посты публикует в канал через Telegram Bot API: фото + подпись.
4. Публикует от старых к новым, чтобы порядок совпадал с Instagram.
5. Обновлённый `posted.json` коммитится обратно в репозиторий — так
   состояние сохраняется между запусками.
6. При самом первом запуске бот ничего не публикует, а просто помечает все
   текущие посты как "уже отправленные" — чтобы не вывалить в канал сразу
   весь архив.

## Настройка

### 1. Создайте бота в Telegram (если ещё нет)

Уже есть — просто возьмите его токен у `@BotFather` (команда `/mybots` →
выбрать бота → API Token).

### 2. Добавьте бота админом в канал

Канал → Administrators → Add Administrator → выбрать бота. Дайте ему право
постить сообщения.

### 3. Узнайте chat_id канала

Если канал публичный — можно использовать `@username_канала` прямо как
`TELEGRAM_CHAT_ID` (с собачкой, например `@my_channel`).

Если канал приватный — нужен числовой ID (обычно вида `-100xxxxxxxxxx`).
Самый простой способ получить: переслать любое сообщение из канала боту
[@JsonDumpBot](https://t.me/JsonDumpBot) или похожему, и посмотреть поле
`chat.id` в ответе.

### 4. Создайте репозиторий на GitHub и залейте туда файлы

Структура:
```
.
├── bot.py
├── posted.json
├── README.md
└── .github/workflows/repost.yml
```

### 5. Добавьте секреты репозитория

Settings → Secrets and variables → Actions → New repository secret:

- `TELEGRAM_BOT_TOKEN` — токен бота от BotFather
- `TELEGRAM_CHAT_ID` — `@username_канала` или числовой `-100...` ID

### 6. (опционально) Задайте RSS_URL как variable

Settings → Secrets and variables → Actions → вкладка **Variables** →
New repository variable:

- `RSS_URL` = `https://rss.app/feeds/pciA3C3UBcPVtJaj.xml`

Если не зададите — в коде есть этот же URL как значение по умолчанию,
так что можно и пропустить этот шаг.

### 7. Первый запуск

Зайдите в Actions → workflow "RSS to Telegram repost" → Run workflow
(вручную, через workflow_dispatch). Первый запуск ничего не постит, только
инициализирует `posted.json` текущими постами — это нормально и ожидаемо.

Дальше workflow будет запускаться сам каждый день по cron (`0 9 * * *` —
09:00 UTC, то есть ~12:00 по Москве). Время можно поменять в
`.github/workflows/repost.yml`, отредактировав строку `cron`.

Если в какой-то момент захочется проверить вручную, не дожидаясь cron —
просто снова нажмите "Run workflow".

## Локальный тест (опционально)

```bash
export TELEGRAM_BOT_TOKEN="..."
export TELEGRAM_CHAT_ID="@my_channel"
python bot.py
```

## Важные нюансы

- **Картинки из Instagram через CDN протухают.** Ссылки на медиа от
  `cdninstagram.com` содержат подписанный токен с ограниченным сроком
  действия. Это не проблема для основного сценария (фид проверяется раз в
  день, ссылки свежие), но если бот не запускался долго и ссылка из старого
  фида устарела — `sendPhoto` может вернуть ошибку. В таком случае пост
  просто не отметится как отправленный и при следующем подходящем запуске
  можно будет вручную скорректировать `posted.json`, либо просто пропустить
  этот конкретный пост.
- **Лимит подписи в Telegram — 1024 символа.** Если текст поста длиннее, он
  будет обрезан (это сделано в коде). Если для вас это критично — могу
  переделать на отправку фото без подписи + отдельным сообщением полный текст.
- **rss.app бесплатный тариф** обычно обновляет фид не мгновенно (может быть
  задержка в десятки минут/часы) — это нормально при проверке раз в день.
